What happens by DEFAULT?

If you don't give any special discovery options, Nmap automatically sends:

Default Discovery Probes:

  • ICMP Echo (like normal ping)
  • TCP SYN to port 443
  • TCP ACK to port 80
  • ICMP timestamp

Discovery Logic

Meaning:
üëâ If a device replies to ANY of these ‚Üí it is alive üëç

On local network:

Nmap uses ARP scan because ARP is 100% accurate locally and faster.

So default is already smart.

Different Levels of Host Discovery

There are 3 main styles of host discovery:

A

List Scan (Only List, Send Nothing)

Command:

nmap -sL <target>

This does:

  • Sends NO packets
  • Just prints list of targets + DNS names

Good for:

  • Checking if your target list is correct
  • Verifying domains
  • Planning scan

But: ‚ùå Does NOT tell who is alive

B

Ping Scan (Find Alive Hosts ‚Äî No Port Scan)

SUPER IMPORTANT

Command:

nmap -sn <target>

This:

  • ‚úîÔ∏è Finds alive machines
  • ‚ùå Does NOT scan ports

Useful for:

  • Network inventory
  • Counting active devices
  • Light, quiet recon

Default probes in -sn:

ICMP Echo TCP SYN 443 TCP ACK 80 ICMP timestamp ARP on local network

üëâ If blocked by firewall, you can customize probes (coming below)

This is what most admins use.

C

No Ping (Assume EVERY Host Is Alive)

Command:

nmap -Pn <target>

Meaning:

  • Skip host discovery completely
  • Treat every IP as alive
  • Go straight to heavy scanning

Useful when:

  • Firewall blocks ping
  • You KNOW targets are alive
  • Internet targets hide ping response

‚ö†Ô∏è Downside:

  • SLOW
  • Scans even dead hosts

Bonus:

nmap -Pn -sn

Skips everything but still runs scripts if needed.

3

Powerful Custom Discovery Options (-P*)

When firewall blocks default ping, we use smarter probes.

1

TCP SYN Ping

nmap -PS80,443 <target>

How it works:

Sends SYN packets. If RST or SYN/ACK returns ‚Üí host alive.

Great when:

  • ICMP blocked
  • SYN allowed
2

TCP ACK Ping

nmap -PA80,443 <target>

Bypasses some firewalls that block SYN but allow ACK.

Tip:

üëâ Many pentesters send BOTH

nmap -PS -PA <target>
3

UDP Ping

nmap -PU <target>

Useful when:

  • TCP firewalled
  • UDP responses more likely

Normally slower.

4

ICMP Ping Types

Normal ping:

nmap -PE <target>

Timestamp ping:

nmap -PP <target>

Address mask ping:

nmap -PM <target>

Great for:

  • Internal networks
  • Admin monitoring

But internet targets often block ICMP.

5

SCTP Ping (rare)

nmap -PY <target>

Used in telecom systems mostly.

6

IP Protocol Ping

nmap -PO <target>

Advanced, rarely needed. Can detect devices even when normal ping fails.

Host Discovery Summary

Nmap uses multiple techniques to determine which hosts are alive before scanning ports. Choose the right method based on your network environment and goals.

List Scan (-sL)

No packets sent, just listing

Ping Scan (-sn)

Finds alive hosts, no port scan

No Ping (-Pn)

Assume all hosts alive
