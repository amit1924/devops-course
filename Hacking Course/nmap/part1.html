1

Check if a device is alive

Ping Scan (No ports yet)

nmap -sn 192.168.1.1

This is called a Ping Scan - it only checks if hosts are online without scanning ports.

Example Output

Starting Nmap 7.94SVN ( https://nmap.org ) at 2025-12-26 10:56 IST
Nmap scan report for amit (192.168.29.133)
Host is up (0.00010s latency).
Nmap done: 1 IP address (1 host up) scanned in 9.58 seconds

Key Points

  • Use -sn flag for host discovery only
  • No port scanning happens with this command
  • Ideal for checking which devices are online on your network
  • Some firewalls may block ping requests, making hosts appear offline

You can also scan a range: nmap -sn 192.168.1.1-50

Service Version Detection

nmap -sV 192.168.1.1

-sV = show service + version (example: Apache, SSH version)

Nmap Command Structure

nmap [HOW to scan] [EXTRA details] [WHO to scan]

Example:

nmap -sV -p 80 192.168.1.1

-sV = HOW (scan with service detection)

-p 80 = EXTRA setting (scan port 80 only)

192.168.1.1 = WHO to scan

Scanning Ranges

Range of IPs:

nmap 192.168.1.1-50

Full Network:

nmap 192.168.1.0/24

Means 192.168.1.1 ‚Üí 192.168.1.255

Port Scanning Basics

NOW we actually check services

Default Simple Scan

nmap 192.168.1.1

Scans the 1000 most common TCP ports.

Port Range Scan

nmap -p 1-1000 192.168.1.1

Scan ports 1 through 1000.

All Ports Scan

nmap -p- 192.168.1.1

Scan all 65535 ports. Can be slow!

Fast Scan (Top Common Ports)

nmap -F 192.168.29.133

Scans only the top 100 most common ports instead of the default 1000. Much faster!

5

Scan Techniques (HOW to scan)

Most Important

"Don't panic üòÑ we'll keep it simple. Most important one in real world."

SYN Scan (Stealth Scan)

sudo nmap -sS 192.168.29.133
fast stealthy MOST used requires sudo/root

This is called a SYN scan - it sends SYN packets and analyzes responses without completing the TCP handshake.

TCP Connect Scan (Non-root)

nmap -sT 192.168.29.133

-sT = TCP Connect Scan. Doesn't need root privileges but is slightly noisier. Works great for learning and when you don't have sudo access.

UDP Scan

nmap -sU 192.168.1.1

UDP scan is slow but important for discovering UDP services like DNS, DHCP, SNMP, etc.

Scanning a Whole Network (CIDR)

Powerful

Format: IP / numberOfBits

Example:

192.168.10.0/24

This scans: 192.168.10.0 ‚Üí 192.168.10.255

Meaning ‚Üí 256 devices

Common CIDR Values

  • /24 most common home network
  • /32 only ONE IP
  • /0 whole internet (DON'T do this üòÜ)

CIDR Quick Reference

CIDR Meaning
/32 only 1 device
/24 256 devices
/16 65,536 devices
/0 whole Internet (crazy üòÖ)

Octet Range Scanning

More Control

Format: 192.168.[RANGE].[RANGE]

CIDR = straight block | Octet ranges = custom pick üòé

Example 1: Skip .0 and .255

192.168.0-255.1-254

Good when avoiding broadcast/network addresses.

Example 2: Pick Specific Subnets

192.168.3-5,7.1

This scans:

192.168.3.1
192.168.4.1
192.168.5.1
192.168.7.1

Wild Super Example

0-255.0-255.13.37

Scans the whole internet but ONLY IPs ending with: 13.37

(used for research sometimes)
