1

-p ‚Äî Choose EXACT Ports to Scan

This is the most powerful option in this section.

Scan Single Port

nmap -p 80 192.168.1.10

Scan Multiple Ports

nmap -p 22,80,443 192.168.1.10

Scan a Range

nmap -p 1-1000 192.168.1.10

Scan ALL 65535 Ports

nmap -p- 192.168.1.10

This is the FULL brutal scan

2

Scan TCP + UDP Together

Normally Nmap only scans TCP. So to scan UDP too:

nmap -sU -sS -p U:53,161,T:22,80 192.168.1.10

Breakdown:

  • U: = UDP ports
  • T: = TCP ports

Example meaning:

  • Scan UDP ports 53, 161
  • Scan TCP ports 22, 80

Without protocol prefix:
üëâ Same ports get added to ALL protocol scans

3

Scan Ports by NAME üòé

Instead of numbers, you can use service names.

Example:

nmap -p ftp,http 192.168.1.10

Or Wildcard:

nmap -p http* 192.168.1.10

This uses nmap-services database.

4

Intelligent Range Port Selection

Meaning:
üëâ Scan ports 1-1024 (well-known ports)

But if you want sequential:

Port Selection Quick Reference

Option Meaning Example
-p Choose ports -p 22,80,443
-p- All 65535 ports -p-
-F Fast 100 ports -F
-r Sequential order -r
--top-ports N Most common N ports --top-ports 100
--exclude-ports Skip certain ports --exclude-ports 80,443

Port Selection Strategy

Choose the right port selection method based on your goals, time constraints, and target environment.

Precise Targeting

Use -p for exact ports

Quick Recon

Use -F or --top-ports

Complete Scan

Use -p- for all ports
