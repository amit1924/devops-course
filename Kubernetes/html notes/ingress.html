üìÑ Your Ingress YAML File

Create this file in your ingress folder: express-ingress.yaml

apiVersion: networking.k8s.io/v1 kind: Ingress metadata: name: express-app-ingress annotations: nginx.ingress.kubernetes.io/rewrite-target: / spec: rules: - host: express.local http: paths: - path: / pathType: Prefix backend: service: name: express-service # ‚Üê MUST match your service name port: number: 3000 # ‚Üê MUST match your service port

Key Points:

  • host: express.local - Your custom domain
  • name: express-service - Must match service name
  • port: 3000 - Must match service port
  • rewrite-target: / - Rewrites paths correctly

Commands to Apply:

# Create the file nano express-ingress.yaml # Paste the YAML (EXACTLY) # Save: Ctrl+O ‚Üí Enter ‚Üí Ctrl+X # Apply it kubectl apply -f express-ingress.yaml # Verify kubectl get ingress

üîç Understanding the Flow

Step What Happens Command to Check Status
1. Browser Request You type http://express.local n/a üü¢ User Action
2. Hosts File /etc/hosts maps to Minikube IP cat /etc/hosts üü¢ DNS Resolution
3. Ingress Controller Nginx receives request kubectl get pods -n kube-system | grep ingress üü¢ Traffic Router
4. Service Routing Routes to express-service kubectl get svc üü¢ Load Balancer
5. Pod Delivery Service sends to pod(s) kubectl get pods üü¢ App Response

üí° Ingress vs NodePort:

NodePort (Before)
  • Access via IP:Port
  • http://192.168.49.2:3xxxx
  • Random port number
  • Not user-friendly
Ingress (Now)
  • Access via domain
  • http://express.local
  • Clean URL routing
  • Production-ready
