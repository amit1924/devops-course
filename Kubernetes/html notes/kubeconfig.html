üß† What is Kubeconfig?

It is a configuration file that tells kubectl:

  • Which cluster to connect to?
    Minikube? AWS? GKE?
  • Which user (credentials) to use?
    Certificates, tokens, etc.
  • Which context (environment)?
    Dev, Staging, Prod?
  • Where is API server located?
    IP address & port

So whenever you run: kubectl get pods

kubectl reads kubeconfig to know: "Where is Kubernetes? Which cluster? Who am I?"

üìç Where is kubeconfig stored?

Default location:

~/.kube/config

In your home directory, under .kube folder

Check it:

# View kubeconfig file cat ~/.kube/config # Or use kubectl kubectl config view

Common Mistakes

  • Wrong context
    Deploying to prod instead of dev
  • Missing kubeconfig
    Cluster not in config file
  • Expired certificates
    Authentication failures
  • Network issues
    Cannot reach API server
