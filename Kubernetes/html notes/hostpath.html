üß† What is hostPath?

üîó

hostPath Definition

hostPath = Mount a folder from your Kubernetes node (Minikube VM / Host) directly into a Pod.

Node Disk (Real machine storage) ‚Üì mounted into Pod / Container folder
üí°

Key Insight

  • ‚úîÔ∏è Data lives on the NODE filesystem
  • Even if Pod deletes ‚Üí data stays üëç
  • But if Node dies ‚Üí data gone ‚ùå

‚úÖ Step 1 ‚Äî Create hostPath Pod

Create the hostpath-pod.yaml file:

apiVersion: v1 kind: Pod metadata: name: hostpath-demo-pod spec: containers: - name: node-app image: busybox command: ["sh", "-c", "while true; do sleep 10; done"] volumeMounts: - name: host-storage mountPath: /host-data volumes: - name: host-storage hostPath: path: /data/host-demo type: DirectoryOrCreate

üìÅ Explanation:

  • path: /data/host-demo - Node directory
  • DirectoryOrCreate - Auto-create if missing
  • mountPath: /host-data - Container mount point
  • busybox - Lightweight container image

üéØ What Happens:

  • Kubernetes creates /data/host-demo on node
  • Mounted into container at /host-data
  • Any file writes go to node filesystem
  • Data survives pod restart/deletion

‚ö†Ô∏è Critical Understanding

hostPath is NOT for production multi-node clusters! Why?

  • ‚Ä¢ If pod moves to another node ‚Üí data lost (different filesystem)
  • ‚Ä¢ No data replication across nodes
  • ‚Ä¢ Security risks (pods can access host filesystem)
  • ‚Ä¢ Node failure = permanent data loss
