üí° First Understand the Concept

A single Pod can contain multiple containers.

üëâ They share the same:

  • Network (same IP, same ports)
    Containers can talk via localhost
  • Storage (if you mount volumes)
    Shared volumes accessible by all containers
  • Lifecycle (start/stop together)
    All containers in pod start and stop together

But they are independent processes running inside the same pod.

üéØ When Do We Need Multi-Container Pods?

Real World Use Case Example Benefit
Logging Sidecar App + log collector (Fluentd) Centralized logs without app changes
Proxy Container App + Nginx reverse proxy SSL termination, load balancing
Helper Container App + init helper (db setup) Initialize before main app starts
Data Processing Main app + worker container Parallel processing in same pod
Monitoring Sidecar App + metrics exporter Real-time monitoring data

Most Common Pattern = Sidecar Pattern

Like a motorcycle sidecar - main container + helper container working together!

‚úÖ Step 1 ‚Äî Create Multi-Container Pod YAML

Inside your Kubernetes folder create: multi-pod.yaml

apiVersion: v1 kind: Pod metadata: name: multi-container-pod spec: containers: - name: express-app image: express-demo:v2 ports: - containerPort: 3000 - name: sidecar-logger image: busybox command: ["sh", "-c", "while true; do echo 'üì¢ Sidecar running inside same pod'; sleep 5; done"]
$ kubectl apply -f multi-pod.yaml
pod/multi-container-pod created
$ kubectl get pods
NAME                      READY  STATUS         RESTARTS  AGE
multi-container-pod   2/2   Running       0        5s

Key Observation:

2/2 in READY column means: 2 containers out of 2 are running! üéâ
