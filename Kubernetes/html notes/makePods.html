üß© Step 2 ‚Äî Create Pod YAML Configuration

Create a file named pod-app.yaml with the following content:

pod-app.yaml
apiVersion: v1 kind: Pod metadata: name: my-node-app labels: app: node-demo spec: containers: - name: node-container image: node:18-alpine command: ["node"] args: ["-e", "require('http').createServer((req,res)=>res.end('Hello from Kubernetes Pod üöÄ')).listen(3000)"] ports: - containerPort: 3000

YAML Breakdown

  • apiVersion: v1 - Kubernetes API version
  • kind: Pod - Resource type
  • metadata.name - Pod name
  • spec.containers - Container definition
  • image: node:18-alpine - Node.js image
  • containerPort: 3000 - Exposed port

Node.js Command Explained

The command creates a simple HTTP server:

require('http').createServer((req, res) => { res.end('Hello from Kubernetes Pod üöÄ') }).listen(3000)

Pro Tip: Alternative Method

You can also create the Pod directly from command line:

$ kubectl run my-node-app --image=node:18-alpine --port=3000 -- node -e "require('http').createServer((req,res)=>res.end('Hello from Kubernetes Pod üöÄ')).listen(3000)"

But using YAML is better for reproducibility and version control!

üåê Step 5 ‚Äî Access the Running App

ü©π If Pod Fails ‚Äî Debug Like a Hero

Check Pod Status

kubectl get pods kubectl describe pod my-node-app kubectl logs my-node-app

Common Issues

  • ImagePullBackOff: Cannot pull image
  • CrashLoopBackOff: Container crashes
  • Pending: No resources available
  • ErrImagePull: Invalid image name

Debug Commands

$ kubectl get events

View cluster events

$ kubectl logs my-node-app --previous

Previous container logs

$ kubectl describe pod my-node-app | grep -A 10 Events

Pod events only

Delete & Retry

# Delete the problematic pod kubectl delete pod my-node-app # Recreate from YAML kubectl apply -f pod-app.yaml

Sometimes the simplest solution is to delete and recreate the Pod. Kubernetes will automatically create a new one.
