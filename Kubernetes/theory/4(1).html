üß† 1Ô∏è‚É£ Resource Requests vs Limits (Deep Understanding)

üü¢ Scheduling Phase (Control Plane) 

resources:
  requests:
    cpu: 0.5
    memory: 256Mi

Scheduler asks:

"Which node has at least 0.5 CPU and 256Mi free?"

If none found ‚Üí Pod = Pending
If cluster autoscaler enabled ‚Üí new node created

üîµ Runtime Phase (Worker Node)

Once scheduled, kubelet:

  • Reserves requested resources
  • Configures runtime to enforce limits

üîπ CPU Behavior

If limit = 1 CPU, container tries to use 2 CPUs?

üëâ It gets throttled

CPU is compressible

üîπ Memory Behavior

If limit = 512Mi, container tries to use 700Mi?

üëâ It gets OOMKilled

Memory is NOT compressible

This CPU vs Memory difference is critical for production

üß† Important Detail

If you define only limits, Kubernetes automatically sets:

You write:

resources:
  limits:
    memory: 512Mi

Kubernetes makes it:

resources:
  requests:
    memory: 512Mi
  limits:
    memory: 512Mi

This can unintentionally reduce bin-packing efficiency - Pod reserves full limit amount even if it only needs less

üß© Multi-Container Pod Scheduling

Scheduler sums ALL container requests in a Pod: 

Pod has:
Container A ‚Üí 200Mi
Container B ‚Üí 300Mi

Pod request = 500Mi total

Scheduler must find node with

‚â• 500Mi available

üü° 2Ô∏è‚É£ Init Containers (Execution Guarantee)

Init Container Characteristics

Run first

Before main containers start

Run sequentially

One after another

Must succeed

Exit code 0 required

Run only once

Per Pod lifecycle

Block main containers

Until init completes

üîÅ Lifecycle Example

0/1

Pod Status: Init:0/1

Init container still running

Main container NOT started yet

Init container exits (exit code 0)

Condition met

Main container starts

Now runs normally

Why This Is Powerful

Instead of putting DB check logic, DNS wait logic, file cloning logic inside main app... You isolate it cleanly. Main app stays simple.

üö® Important Behavior

If init container fails:

Pod stays in failed init state

Main container never starts
