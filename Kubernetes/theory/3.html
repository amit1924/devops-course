kubeconfig File Explained

Clusters

List of Kubernetes clusters kubectl knows about

  • Name
  • API endpoint
  • Certificate

Users

Credentials for authentication

  • dev, ops, admin
  • Certificates/keys
  • X.509 certificates

Contexts

Cluster + User combination

  • Links cluster + user
  • Friendly name
  • Quick switching

kubeconfig Example 

apiVersion: v1
kind: Config
# CLUSTERS
clusters:
- name: shield  ‚Üê Friendly name
  cluster:
    server: https://192.168.1.77:8443  ‚Üê API endpoint
    certificate-authority-data: LS0tLS1CRUdJTiBDRVJ...

# USERS
users:
- name: coulson  ‚Üê User credentials
  user:
    client-certificate-data: LS0tLS1CRUdJTiBDRV...
    client-key-data: LS0tLS1CRUdJTiBFQyB...

# CONTEXTS
contexts:
- name: director  ‚Üê Context name
  context:
    cluster: shield      ‚Üê Use this cluster
    user: coulson        ‚Üê As this user

# CURRENT CONTEXT
current-context: director  ‚Üê Active context
