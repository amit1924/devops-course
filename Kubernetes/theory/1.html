Chapter 1: Kubernetes Primer

Core Concepts & Fundamentals

What is Kubernetes?

Simple Definition: Automatically runs and manages container-based applications

Book's Jargon: "Orchestrator of containerized cloud-native microservices apps"

Better Translation: Runs/manages container-based app components with auto-scale, heal, update

Key Components Explained

1. Orchestration

Automatic management of multiple containers

Like a conductor managing an orchestra

2. Containerization

Lightweight application packaging

Containers vs Virtual Machines

Feature Virtual Machines Containers
OS Full OS inside Shares host OS
Size Heavy (GBs) Light (MBs)
Start Time Minutes Seconds
Real-world Analogy Renting full apartment Renting a room in shared apartment

Cloud-Native Explained

❌ NOT Cloud-Native

Uploading old monolith to AWS EC2 = Just "cloud-hosted"

✅ Cloud-Native Means

  • Auto-scaling
  • Self-healing
  • Zero-downtime updates
  • Distributed design

Microservices Pattern

Old Way: Monolith

One big app with everything → If one part breaks, everything affected

New Way: Microservices

Small independent services → Scale/update independently

Example: Black Friday → Scale only cart & catalog services, leave auth untouched

Chapter 2: Kubernetes Principles of Operation

Architecture & Core Components

Kubernetes Cluster Architecture

Control Plane Nodes

The Brain - Makes decisions, manages cluster

  • Scheduling & orchestration
  • Self-healing operations
  • Auto-scaling
  • Rolling updates

Production Rule: Use 3 or 5 nodes for HA (odd numbers for quorum)

Worker Nodes

The Muscles - Run actual containers/workloads

  • Execute containerized apps
  • Provide CPU, memory, storage
  • Run microservices

Best Practice: Keep user apps on workers, control plane focuses on management

Control Plane Components

API Server

Front-end to Kubernetes. All requests go through here.

REST API • HTTPS • Authentication • Authorization
Cluster Store (etcd)

Holds desired state of all apps & components.

Distributed DB • RAFT consensus • Only stateful part
Controller Manager

Spawns/manages controllers that ensure desired state.

Watch loops • Reconciliation • Self-healing

Why Kubernetes Matters - Strategic View

1. Infrastructure Abstraction

Hides cloud complexity like OS hides hardware complexity

You say: "Run 3 copies of my app"
Kubernetes decides: Which machine, zone, network, storage

2. Multi-Cloud & Migration

Reduces vendor lock-in, enables hybrid cloud

Today: AWS → Tomorrow: Azure → Next Year: On-premise

Your app talks to Kubernetes API, not cloud-specific APIs

Golden Mental Model

1

Kubernetes = Operating System of the Cloud

Linux schedules processes → Kubernetes schedules containers

2

Clear Layered Stack

Cloud → VMs → Kubernetes → Container Runtime → Containers → Microservices → App
3

Control Plane = CEO/Managers

Worker Nodes = Employees doing actual work
