üß† Important Rule

Inside one Namespace:

Names must be unique

Cannot have two "express-app" in finance

Across different Namespaces:

Same names are allowed

Both finance and hr can have "express-app"

This is valid: finance/express-app AND hr/express-app

üîê Isolation Level

What Namespaces Isolate

Resource names

Can have same names in different namespaces

RBAC permissions

Different access controls per namespace

Network policies

Control traffic between namespaces

Resource quotas

Limit CPU/memory per namespace

‚ö†Ô∏è Important Limitation

Namespaces provide logical isolation, NOT physical isolation

Finance cannot accidentally delete HR resources:

kubectl delete deployment express-app -n finance

Only deletes finance deployment

But important:

They still run on same physical nodes!

If a node crashes, both namespaces are affected

üß† Why Namespaces Are Powerful

Without Namespaces

  • All apps dumped into "default" namespace
  • Name conflicts happen frequently
  • Hard to manage and organize
  • No separation between teams
  • No per-team resource quotas

With Namespaces

  • Clean separation of concerns
  • Resource quotas per namespace:
    • Finance: max 4 CPU
    HR: max 2 CPU
    DevOps: max 8 CPU
  • Team-specific RBAC policies
  • Clean resource organization

üß† Solution: Set Default Namespace for kubectl

1

Tell kubectl your namespace

kubectl config set-context --current --namespace finance

"From now on, assume I am working in finance namespace"

2

Now simple commands work

kubectl get pods

Now automatically means: kubectl get pods -n finance

kubectl get svc

Now automatically means: kubectl get svc -n finance

No more typing -n every single time!

üöÄ Deploying to a Namespace

üü¢ Imperative Way (Quick Command)

Quick but you need to remember -n flag every time

üîµ Declarative Way (Best Practice)

  name: express-api
  namespace: finance  ‚Üê Specify in YAML

Now when you run:

Clean, self-documenting, no flags to remember

üß† Important Gotcha

If you delete the namespace while kubectl context points to it:

kubectl get pods

Returns nothing (namespace doesn't exist)

You must reset context:

kubectl config set-context --current --namespace default
