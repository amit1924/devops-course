PART 1: Creating the Policy Document

Breaking Down Every Word

1. nano

Meaning: Terminal text editor program

Real-world analogy:

Windows ‚Üí Notepad
macOS ‚Üí TextEdit

2. s3-read-only.json

Meaning: File name you're creating

Filename breakdown:

s3-read-only Purpose/description
.json File format (JSON = JavaScript Object Notation)

üí° What happens when you run this?

  1. Opens the nano text editor
  2. Creates/edits file named s3-read-only.json
  3. You can now type JSON content
  4. Save with Ctrl+O ‚Üí Press Enter
  5. Exit with Ctrl+X

Example JSON Content 

{
  "Version": "2012-10-17", // IAM policy version
  "Statement": [
    {
      "Effect": "Allow", // Allow or Deny
      "Action": [ // What actions are allowed
        "s3:ListAllMyBuckets", // List all buckets
        "s3:ListBucket", // List objects in a bucket
        "s3:GetObject" // Read/download objects
      ],
      "Resource": "*" // Which resources (all S3 resources)
    }
  ]
}

JSON Structure Meaning

This is a permission rulebook that says:

"Allow read-only access to S3"

Key Permissions

List buckets
List objects
Read objects

What's NOT Allowed

Upload files
Delete files
Create buckets

PART 2: Creating the IAM Policy

aws iam create-policy \
  --policy-name S3ReadOnlyPolicy \
  --policy-document file://s3-read-only.json \
  --endpoint-url=http://localhost:4566

Word-by-Word Breakdown

1. aws

Meaning: AWS Command Line Interface

The tool that talks to AWS (or LocalStack)

2. iam

Meaning: Identity and Access Management

Service for managing users, roles, permissions

3. create-policy

Meaning: Command to create a new IAM policy

Think of it as:

"Register this rulebook in the system"

4. --policy-name

Meaning: Parameter flag for policy name

Tells AWS: "Name this policy..."

S3ReadOnlyPolicy

Just a label (no spaces allowed)

5. S3ReadOnlyPolicy

Meaning: The actual policy name

Other valid examples:

EC2FullAccess LambdaExecuteOnly S3UploadAccess

6. --policy-document

Meaning: Parameter flag for rules file

Tells AWS: "Here are the rules..."

7. file://s3-read-only.json

Meaning: Path to the JSON file

‚ö†Ô∏è Important:

Must use file:// prefix

Without it, AWS CLI thinks it's inline JSON

8. --endpoint-url=http://localhost:4566

Meaning: Send this command to LocalStack

Without this:

Goes to real AWS (needs internet, costs money)

With this:

Goes to LocalStack (free, local, no internet)

üéØ What Happens After This Command

  1. LocalStack reads the JSON file
  2. Creates a policy object internally
  3. Generates an ARN (Amazon Resource Name)
  4. Returns something like:
{
  "Policy": {
    "PolicyName": "S3ReadOnlyPolicy",
    "PolicyId": "ANPA...",
    "Arn": "arn:aws:iam::000000000000:policy/S3ReadOnlyPolicy",
    "CreateDate": "2026-01-31T11:22:00Z"
  }
}

PART 3: Attaching Policy to User

aws iam attach-user-policy \
  --user-name dev-user \
  --policy-arn arn:aws:iam::000000000000:policy/S3ReadOnlyPolicy \
  --endpoint-url=http://localhost:4566

This is Where Magic Happens ‚ú®

Real-World Analogy

Step 1: You wrote a rulebook (policy)

The JSON file with S3 read-only rules

Step 2: You registered it (create-policy)

Filed it in the system, got an ARN

Step 3: You give it to employee (attach-user-policy)

Now dev-user actually gets the permissions

Word-by-Word Breakdown

1. attach-user-policy

Meaning: Link a policy to a user

The action of giving permissions

2. --user-name dev-user

Meaning: Which user gets the policy

Remember you created this earlier:

aws iam create-user --user-name dev-user

3. --policy-arn

Meaning: Specify which policy to attach

‚ö†Ô∏è Critical Understanding:

AWS uses ARN, not policy name, for identification

4. arn:aws:iam::000000000000:policy/S3ReadOnlyPolicy

Meaning: Amazon Resource Name (unique identifier)

ARN Breakdown

arn aws iam 000000000000 policy S3ReadOnlyPolicy
