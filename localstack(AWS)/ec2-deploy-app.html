4.1

Create Your "EC2 Server" (Docker)

Docker Command
docker run -it --name ec2-app -p 3000:3000 ubuntu bash

Parameter Breakdown:

  • -it Interactive terminal
  • --name ec2-app Your server name
  • -p 3000:3000 Public app port mapping
  • ubuntu EC2 OS (Amazon Linux equivalent)
  • bash Start with shell

Real EC2 Mindset:

You just:

  • Launched a new virtual server
  • Opened port 3000 to public
  • Selected Ubuntu OS
  • Got terminal access

This is exactly EC2 launch workflow

4.4

Install Node.js (Like Production)

Production Installation
curl -fsSL https://deb.nodesource.com/setup_18.x | bash - apt install -y nodejs

Verify Installation:

node -v npm -v

Should show Node.js version (v18.x.x)

Why This Way?

  • Uses official NodeSource repository
  • Gets latest stable Node.js 18
  • Includes npm automatically
  • Production-ready setup

üéØ Production Note:

This is how you'd install Node.js on real EC2 for production apps.

Different from just apt install nodejs (which gives older version)

4.8

Production Mindset: Run App in Background

‚ö†Ô∏è Reality Check:

On real EC2:

  • We don't keep terminal open forever
  • App must survive terminal disconnect
  • Need process management
Install PM2 (Production Process Manager)
npm install -g pm2

Run App with PM2:

pm2 start index.js pm2 save

Check Status:

pm2 status pm2 logs

‚úÖ Production Benefits:

App survives:

  • Terminal close
  • SSH disconnect
  • Server reboot

Like real EC2:

  • Auto-restart on crash
  • Log management
  • Process monitoring
4.9

Stop/Start Like Real EC2

Exit Container:

exit

Returns to host terminal (like logging out of EC2)

Stop Server:

docker stop ec2-app

Like EC2 "Stop Instance"

Start Server Again:

docker start ec2-app

Like EC2 "Start Instance"

App still works after restart

Same as EC2 reboot behavior
