6.2

Step 2 ‚Äî Configure Reverse Proxy

Edit NGINX Default Site Configuration
nano /etc/nginx/sites-available/default
/etc/nginx/sites-available/default 
server {
    listen 80;

    server_name localhost;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}

üîß Configuration Breakdown:

  • listen 80 Listens on standard HTTP port
  • server_name localhost Server name (localhost for local)
  • proxy_pass http://localhost:3000 Forwards requests to your app
  • proxy_set_header Passes headers for WebSocket support

6.6

Step 6 ‚Äî Restart App & NGINX (Production Mindset)

1. Stop Application

pm2 stop index.js

Gracefully stops your Node.js application

2. Start Application

pm2 start index.js

Starts application with latest code

3. Restart NGINX

service nginx restart

Reloads NGINX configuration

üîÑ Why Restart NGINX?

  • Applies configuration changes
  • Graceful reload without dropping connections
  • Zero-downtime deployment ready
  • Standard production practice

üéØ Production Alternatives:

service nginx reload

Graceful reload (keeps connections)

systemctl restart nginx

Systemd restart
