Docker Basics & Concepts

What is Docker?

Docker is a platform for developing, shipping, and running applications in containers. Containers package an application with all its dependencies, ensuring it runs uniformly across different environments.

Key Components

  • Docker Engine: The core product that creates and runs containers
  • Docker Image: A read-only template with instructions for creating a container
  • Docker Container: A runnable instance of an image
  • Docker Registry: A repository for Docker images (Docker Hub)
  • Dockerfile: A text file with commands to build an image

How Docker Works

  • Uses containerization technology
  • Shares the host OS kernel
  • Isolates applications in containers
  • Uses Union File Systems for efficiency
  • Provides consistent environments

Key Docker Concepts

Container

A lightweight, standalone, executable package that includes everything needed to run a piece of software.

Image

A read-only template with instructions for creating a Docker container. Built from a Dockerfile.

Dockerfile

A text document that contains all the commands a user could call on the command line to assemble an image.

Volume

Persistent data storage that exists outside a container's lifecycle, used to share data between containers or persist data.

Essential Docker Commands

Container Management

Run a container

docker run [options] image [command] [args]

Creates and starts a container from an image.

# Example: Run an Nginx container
docker run -d -p 80:80 --name my-nginx nginx

List containers

docker ps [options]

Lists running containers. Use -a to show all containers.

docker ps -a
docker ps --filter "status=exited"

Stop/Start containers

docker stop container
docker start container
docker restart container

Manage container lifecycle.

docker stop my-nginx
docker start my-nginx

Image Management

Build an image

docker build [options] path

Builds an image from a Dockerfile.

docker build -t my-app:1.0 .
docker build -f Dockerfile.prod -t my-app:prod .

Manage images

docker images
docker rmi image
docker pull image
docker push image

List, remove, pull, and push images.

docker pull ubuntu:20.04
docker rmi ubuntu:18.04

Advanced Docker Commands

Docker Compose

docker-compose up [options]
docker-compose down
docker-compose ps

Manage multi-container applications.

docker-compose up -d
docker-compose down -v

Logs and Inspection

docker logs container
docker exec [options] container command
docker inspect container

Check container logs, execute commands, and inspect details.

docker logs -f my-nginx
docker exec -it my-nginx bash

Docker Real-World Examples

Dockerfile Example

# Dockerfile for a Node.js application

FROM node:14-alpine

WORKDIR /app

COPY package*.json ./
RUN npm install

COPY . .

EXPOSE 3000

CMD ["npm", "start"]

Build and Run

# Build the image
docker build -t my-node-app .

# Run the container
docker run -d -p 3000:3000 --name node-app my-node-app

Docker Compose Example

# docker-compose.yml
version: '3.8'
services:
  web:
    build: .
    ports:
      - "3000:3000"
  db:
    image: postgres:13
    environment:
      POSTGRES_PASSWORD: example

Kubernetes Basics & Concepts

What is Kubernetes?

Kubernetes (K8s) is an open-source container orchestration platform that automates deploying, scaling, and managing containerized applications.

Key Components

  • Pod: Smallest deployable unit (one or more containers)
  • Node: Worker machine (VM or physical)
  • Cluster: Set of nodes running containerized apps
  • Deployment: Manages pods and ReplicaSets
  • Service: Network endpoint for pods
  • Namespace: Virtual cluster within a physical cluster

Architecture

  • Control Plane: Manages the cluster (API Server, etcd, Scheduler)
  • Worker Nodes: Run containerized applications
  • kubelet: Agent running on each node
  • kube-proxy: Network proxy on each node
  • Container Runtime: Docker, containerd, etc.

Key Kubernetes Concepts

Pod

Smallest deployable unit in Kubernetes, containing one or more containers that share storage and network.

Deployment

Manages a replicated application, ensuring the desired number of pods are running and available.

Service

Abstract way to expose an application running on a set of pods as a network service.

Namespace

Virtual cluster within a Kubernetes cluster, used to organize resources and divide cluster resources.

Essential kubectl Commands

Basic Resource Management

Get resources

kubectl get resource_type [options]

List resources like pods, deployments, services.

kubectl get pods
kubectl get pods -n default
kubectl get pods --all-namespaces
kubectl get deployments,services

Create resources

kubectl create [options]
kubectl apply -f file.yaml

Create resources from file or command line.

kubectl create deployment nginx --image=nginx
kubectl apply -f deployment.yaml

Describe resources

kubectl describe resource_type/resource_name

Show detailed information about a resource.

kubectl describe pod/my-pod
kubectl describe deployment/nginx

Advanced kubectl Commands

Logs and debugging

kubectl logs pod_name [options]
kubectl exec [options] pod_name -- command

Check pod logs and execute commands in pods.

kubectl logs my-pod -f
kubectl exec -it my-pod -- bash

Port forwarding

kubectl port-forward pod_name local_port:pod_port

Forward local port to a pod.

kubectl port-forward my-pod 8080:80

Managing deployments

kubectl rollout status deployment/deployment_name
kubectl scale deployment/deployment_name --replicas=3
kubectl set image deployment/deployment_name container=image:tag

Monitor, scale, and update deployments.

Kubernetes Real-World Examples

Deployment YAML Example

# nginx-deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
  labels:
    app: nginx
spec:
  replicas: 3
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: nginx:1.14.2
        ports:
        - containerPort: 80

Service YAML Example

# nginx-service.yaml
apiVersion: v1
kind: Service
metadata:
  name: nginx-service
spec:
  selector:
    app: nginx
  ports:
    - protocol: TCP
      port: 80
      targetPort: 80
  type: LoadBalancer

Commands to Apply

# Create the deployment
kubectl apply -f nginx-deployment.yaml

# Create the service
kubectl apply -f nginx-service.yaml

# Check status
kubectl get deployments
kubectl get pods
kubectl get services

Docker & Kubernetes Glossary

Docker Terms

Docker Hub

Cloud-based registry service for sharing Docker images.

Docker Compose

Tool for defining and running multi-container Docker applications.

Docker Swarm

Docker's native clustering and orchestration tool.

Union File System

Layered file system that Docker uses for images.

Kubernetes Terms

ConfigMap

API object used to store non-confidential data in key-value pairs.

Secret

Object that contains a small amount of sensitive data.

Ingress

API object that manages external access to services.

StatefulSet

Manages stateful applications, maintaining pod identity.
