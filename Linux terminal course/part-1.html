218

126-134

echo

Displays text or variables to terminal or redirects to files.

$ echo "hi"
hi

$ echo "Amit" > a.txt # Overwrites file
$ echo "More text" >> a.txt # Appends to file
$ name="Amit"
$ echo $name
Amit
Redirection Operators:
• > - Overwrites file (creates if doesn't exist)
• >> - Appends to file
• 2> - Redirects stderr (error output)
• &> - Redirects both stdout and stderr

Permissions & Sudo

sudo

Execute command as superuser (root). Short for "SuperUser DO".

$ sudo apt update # Update package list
$ sudo apt install tree # Install package
$ sudo -h # Show help
$ sudo ~ # Execute command in home as root
$ sudo -i # Login as root interactively
$ sudo su # Switch to root user
Common sudo patterns:
• sudo !! - Run previous command with sudo
• sudo -u username command - Run as specific user
• sudo -k - Clear cached credentials
• sudo -v - Refresh credentials

Searching & Finding

find

Searches for files and directories based on various criteria.

$ sudo find / -name transfer-dynamic 2>/dev/null
/usr/local/bin/transfer-dynamic

$ sudo find / -name transfer-dynamic
find: '/run/user/1000/doc': Permission denied
find: '/run/user/1000/gvfs': Permission denied
find: '/proc/40599/task/40599/net': Invalid argument
find: '/proc/40599/net': Invalid argument
/usr/local/bin/transfer-dynamic

$ find . -name "*.txt"
./a.txt
./b.txt
./test1/a.txt

$ sudo find / -name "*.txt" 2>/dev/null
# Search entire system, hide errors
$ find ~ -name "*.txt"
# Search only in home directory
Common find patterns:
• find . -type f -name "*.txt" - Only files
• find . -type d -name "dir*" - Only directories
• find . -mtime -7 - Modified in last 7 days
• find . -size +100M - Files larger than 100MB
• 2>/dev/null - Redirects stderr to /dev/null (hides errors)

Shell Variables

Variables store data for shell scripts and commands.

$ echo "hi"
hi

$ echo $date
# Empty if date variable not set

$ date=$(date) # Command substitution
$ echo date
date # Literal string

$ echo $date
Mon Dec 22 12:42:15 UTC 2025

$ name="Amit"
$ echo $name
Amit

$ echo "$(uname)"
Linux # Command substitution in string
Variable Rules:
• No spaces around = sign
• Use $ to access variable value
• Use $(command) for command substitution
• "$var" preserves spaces, $var doesn't

Best Practices & Pro Tips

Command Chaining

  • ; - Run sequentially: cmd1; cmd2
  • && - Run if previous succeeds: cmd1 && cmd2
  • || - Run if previous fails: cmd1 || cmd2
  • | - Pipe output: cmd1 | cmd2

Dangerous Commands

  • rm -rf / - Deletes EVERYTHING!
  • :(){ :|:& };: - Fork bomb
  • mkfs.ext4 /dev/sda - Formats drive
  • dd if=/dev/zero of=/dev/sda - Wipes drive

Productivity Tips

  • Tab - Auto-complete
  • ↑/↓ - Navigate history
  • Ctrl+R - Search history
  • Ctrl+C - Cancel command
  • Ctrl+D - Exit shell
