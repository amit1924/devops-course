File Statistics & Analysis

New

wc - Word Count

Counts lines, words, and characters in files.

$ wc -l /etc/passwd
50 /etc/passwd
wc Options:
• -l - Count lines only
• -w - Count words only
• -c - Count characters only
• -m - Count characters (multi-byte)

Output Interpretation:
"50 /etc/passwd" means 50 lines in the file
Each line = 1 user account on the system
New

cut - Extract Sections

Cuts out selected portions from each line of files.

$ cut -d: -f1,7 /etc/passwd
root:/bin/bash
daemon:/usr/sbin/nologin
bin:/usr/sbin/nologin
sys:/usr/sbin/nologin
sync:/bin/sync
games:/usr/sbin/nologin
man:/usr/sbin/nologin
...
amit:/bin/bash
amish:/bin/sh
cut Options Explained:
• -d: - Use : as delimiter (field separator)
• -f1,7 - Select fields 1 and 7

/etc/passwd Fields:
Field 1: Username
Field 2: Password placeholder (x means password in /etc/shadow)
Field 3: UID (User ID)
Field 4: GID (Group ID)
Field 5: GECOS (User info/comment)
Field 6: Home directory
Field 7: Shell

chmod - Changing Permissions

$ chmod +u-x script.sh # Incorrect syntax!
$ ./script.sh
zsh: permission denied: ./script.sh

$ chmod +x script.sh # Correct syntax - add execute
$ ./script.sh
hello world
Hello, Alice!
Number: 1
Number: 2
...
Enter your favorite color:
Symbolic Method
chmod [ugoa][+-=][rwx] file

• u = user (owner)
• g = group
• o = others
• a = all (default)
• + = add permission
• - = remove permission
• = = set exact permission

Examples:
chmod u+x file
chmod g-w file
chmod o=r file
chmod a+x file
Numeric (Octal) Method
chmod XXX file

• 4 = Read (r)
• 2 = Write (w)
• 1 = Execute (x)

Calculate:
7 = 4+2+1 = rwx
6 = 4+2 = rw-
5 = 4+1 = r-x
4 = 4 = r--
0 = ---

Examples:
chmod 755 file
chmod 644 file
chmod 700 file

Folder vs File Permissions

On Files:
• r = Read file contents
• w = Modify/delete file
• x = Execute as program
On Folders:
• r = List folder contents (ls)
• w = Create/delete files in folder
• x = Enter folder (cd)
Important: To access a folder, you need x permission!
A folder with r-- permission is useless - you can see file names but not access them.

User Management

Creating & Modifying Users

$ sudo useradd amish
$ passwd amish
passwd: You may not view or modify password information for amish.

$ sudo passwd amish
New password:
Retype new password:
passwd: password updated successfully

$ sudo usermod -aG sudo amish
$ groups amish
amish : amish sudo
Common User Commands:
• useradd - Create new user
• usermod - Modify existing user
• userdel - Delete user
• passwd - Change password
• groups - Show user's groups

Important Flags:
• -aG = Append to Group (don't remove from others)
• -m = Create home directory (with useradd)
• -s = Set shell (e.g., /bin/bash)

Home Directory Management

$ ls -ld /home/amish
"/home/amish": No such file or directory (os error 2)

$ sudo mkdir /home/amish
$ sudo chown amish:amish /home/amish
$ ls -ld /home/amish
drwxr-xr-x - amish 22 Dec 14:58 /home/amish

$ ls -ld /home/amish
drwxr-x--- - amish 22 Dec 14:33 /home/amish
Ownership Commands:
• chown - Change file owner
• chgrp - Change file group

Syntax:
chown user:group file
chown user file
chown :group file

Default Home Permissions:
750 (drwxr-x---) = Owner full, group enter, others nothing

Quick Memory Trick

Action Command
Create user adduser username
Give sudo usermod -aG sudo username
Switch user su - username
Delete user userdel username
Delete user + home userdel -r username

Danger Zone - User Deletion

$ sudo userdel -r amish
$ sudo rm -rf /home/amish

Warning: userdel -r deletes user AND home directory!
rm -rf is irreversible deletion.

Why Two Files?

Historical Context: In early UNIX, /etc/passwd contained encrypted passwords.
Problem: Anyone could read /etc/passwd and try to crack passwords.
Solution: Move passwords to /etc/shadow (root-only access).
Result: /etc/passwd now has 'x' indicating password is in /etc/shadow.

Practice Exercises

Exercise 1: User Analysis

  1. Count how many users have /bin/bash as shell
  2. List all users with UID above 1000
  3. Find which users have no password (!! in shadow)
  4. Check when each user last changed password
Hint: Use cut, grep, wc together

Exercise 2: Permission Testing

  1. Create a script that needs execute permission
  2. Try running it without permission
  3. Add execute permission using symbolic method
  4. Remove write permission from group
  5. Set permissions to 751 using numeric method
Hint: Use chmod u+x and chmod 751

Exercise 3: User Management

  1. Create a test user "demo" with home directory
  2. Set password for demo user
  3. Add demo user to "sudo" group
  4. Change demo's shell to /bin/bash
  5. Clean up: delete demo user and home
Hint: Use useradd -m and usermod -aG
