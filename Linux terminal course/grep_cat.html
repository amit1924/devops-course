grep - Theory & Fundamentals

Global Regular Expression Print

What is grep?

grep is a command-line utility for searching plain-text data sets for lines that match a regular expression pattern. It's one of the most powerful and frequently used commands in Unix-like operating systems.

Basic Syntax

grep [options] pattern [file...]

Common Use Cases

  • • Search for specific text in files
  • • Filter command output
  • • Find patterns in log files
  • • Extract specific lines

How grep Works Internally

1
Input Reading

grep reads input line by line from files or stdin

2
Pattern Matching

Compares each line against the specified regular expression

3
Output Filtering

Prints only lines that match the pattern (by default)

The grep Family

grep

Basic grep with BRE (Basic Regular Expressions)

egrep / grep -E

Extended Regular Expressions (ERE)

fgrep / grep -F

Fixed strings (no regex, faster)

rgrep / grep -r

Recursive search in directories

grep - Complete Examples

Basic Search Examples

1. Simple Text Search

$ grep "error" logfile.txt
[2023-12-22 10:30:15] ERROR: Connection failed
[2023-12-22 11:45:22] error 404 not found
FATAL ERROR: System crash detected

Case-sensitive search for "error"

2. Case-Insensitive Search

$ grep -i "error" logfile.txt
[2023-12-22 10:30:15] ERROR: Connection failed
[2023-12-22 11:45:22] error 404 not found
FATAL ERROR: System crash detected
Warning: Minor Error in processing

Matches ERROR, error, Error, etc.

3. Search Multiple Files

$ grep "TODO" *.py *.js
app.py:TODO: Add error handling here
utils.py:TODO: Optimize this function
script.js:// TODO: Implement validation

Searches in all .py and .js files

Advanced grep Examples

4. Invert Match (Exclude)

$ grep -v "debug" app.log
[INFO] Application started
[ERROR] Database connection failed
[WARN] High memory usage
# DEBUG lines are excluded

Shows lines NOT containing "debug"

5. Count Matches

$ grep -c "success" transactions.log
42

$ grep -c -v "failed" transactions.log
158

Counts matching/non-matching lines

6. Line Numbers

$ grep -n "function" script.js
15:function calculateTotal() {
42:function validateInput(input) {
78:function init() {

Shows line numbers with matches

7. Recursive Search

$ grep -r "TODO" .
./src/main.py:TODO: Refactor this module
./src/utils.py:TODO: Add logging
./tests/test_api.py:TODO: Write more tests
./docs/README.md:TODO: Update installation guide

Recursively searches all files in current directory

Regular Expression Examples

8. Basic Regex Patterns

$ grep -E "^\[ERROR\]" logfile.txt
[ERROR]: Database connection failed
[ERROR]: File not found
# Matches lines starting with [ERROR]

$ grep -E "\.(py|js|java)$" files.txt
app.py
script.js
Main.java
# Matches .py, .js, or .java extensions

9. Character Classes

$ grep -E "[0-9]{3}-[0-9]{3}-[0-9]{4}" contacts.txt
John: 555-123-4567
Jane: 888-999-0000
# Matches phone number patterns

$ grep -E "[A-Z][a-z]+ [A-Z][a-z]+" names.txt
John Smith
Alice Johnson
# Matches "First Last" name format

10. Word Boundaries

$ grep -w "run" commands.txt
Please run the script
Don't run with sudo
# Matches "run" but not "running" or "prune"

cat - Theory & Fundamentals

Concatenate and Display Files

What is cat?

cat (short for concatenate) is a standard Unix utility that reads files sequentially, writing them to standard output. Despite its simple name, it's incredibly versatile.

Basic Syntax

cat [options] [file...]

Primary Functions

  • • Display file contents
  • • Concatenate multiple files
  • • Create new files
  • • Copy/append file content

How cat Works Internally

1
File Reading

Reads files in order specified (or stdin if no files)

2
Buffer Management

Reads data in chunks for efficiency

3
Output Writing

Writes content to stdout (screen) by default

4
Error Handling

Continues with next file if one fails (unless -b)

Common Use Cases

File Viewing

Quickly view contents of small files

File Creation

Create new files with user input

File Concatenation

Combine multiple files into one

Data Piping

Pass file content to other commands

cat - Complete Examples

Basic File Operations

1. Display File Contents

$ cat file.txt
Line 1: Hello World
Line 2: This is a test file
Line 3: Created with cat command
Line 4: End of file

Simple file viewing - most common use

2. Create New File

$ cat > newfile.txt
# Type your content, then Ctrl+D to save
This is line 1
This is line 2
^D

$ cat newfile.txt
This is line 1
This is line 2

Creates file with keyboard input (Ctrl+D to finish)

3. Append to File

$ cat >> existing.txt
# Adds to end of file
Additional line 1
Additional line 2
^D

Appends content without overwriting

Advanced cat Operations

4. Concatenate Multiple Files

$ cat file1.txt file2.txt file3.txt > combined.txt
# Creates combined.txt with all three files

$ cat combined.txt
=== Contents of file1.txt ===
=== Contents of file2.txt ===
=== Contents of file3.txt ===

Merges files in specified order

5. Number Lines

$ cat -n data.txt
1 Apple
2 Banana
3 Cherry
4 Date
5 Elderberry

$ cat -b code.py
1 def hello():
2 print("Hello")

3 def world():
4 print("World")

-n numbers all lines, -b numbers non-blank lines

6. Show Non-Printable Characters

$ cat -A special.txt
Line 1$
Line 2^I(tab character)$
Line 3^M$
End of file$

-A shows $ for newlines, ^I for tabs, ^M for carriage returns

7. Squeeze Blank Lines

$ cat -s messy.txt
Line 1

Line 2

Line 3
# Multiple blank lines become single blank lines

Reduces multiple consecutive blank lines to single blank lines

Practical Use Cases

8. Create Configuration File

$ cat > config.env << 'EOF'
DATABASE_HOST=localhost
DATABASE_PORT=5432
DATABASE_USER=admin
DATABASE_PASS=secret123
DEBUG=true
EOF

$ cat config.env
DATABASE_HOST=localhost
DATABASE_PORT=5432
DATABASE_USER=admin
DATABASE_PASS=secret123
DEBUG=true

Here-document syntax for multi-line input

9. Copy File Content

$ cat source.txt > destination.txt
$ cat image.jpg > backup.jpg
# Works with binary files too!

Simple file copy (though cp is better for this)

10. View Multiple Files

$ cat file1.txt file2.txt
=== Contents of file1.txt ===
Line 1 from file1
Line 2 from file1
=== Contents of file2.txt ===
Line 1 from file2
Line 2 from file2

View multiple files sequentially without concatenating

grep vs cat: Comparison & When to Use

Feature grep cat
Primary Purpose Search/filter text Display/concatenate files
Output Only matching lines Entire file content
Pattern Matching ✓ Yes (regex) ✗ No
File Creation ✗ No ✓ Yes (with >)
Speed for Large Files Fast (stops early) Slower (reads all)
Best For Searching, filtering Viewing, combining
Common Flags -i, -v, -n, -r, -c -n, -b, -A, -s

Which One to Use?

Use grep when:

  • • Searching for specific text in files
  • • Filtering command output
  • • Finding patterns in logs
  • • Extracting specific lines
  • • Counting occurrences

Use cat when:

  • • Viewing complete file contents
  • • Creating small text files
  • • Concatenating multiple files
  • • Appending to files
  • • Piping file content to other commands

Use Both Together:

$ cat largefile.txt | grep "error" | head -20
$ grep "TODO" *.py | cat -n
$ cat access.log | grep "404" | wc -l

Combine strengths: cat feeds data, grep filters it

Advanced Usage & Pro Tips

Advanced grep Techniques

Context Lines

$ grep -B2 -A2 "crash" system.log
[INFO] Memory usage: 45%
[WARN] Memory usage: 85%
[ERROR] System crash detected
[INFO] Attempting recovery
[INFO] Recovery successful

-B2: 2 lines Before match
-A2: 2 lines After match
-C3: 3 lines Context (both sides)

Binary File Search

$ grep -a "text" binaryfile.bin
$ grep -I "TODO" *
# -a: treat binary as text
# -I: skip binary files

Colored Output

$ grep --color=auto "pattern" file.txt
$ alias grep='grep --color=auto'
# Add to ~/.bashrc for permanent colored grep

Performance Tips

$ grep -F "fixed.string" largefile.txt
$ grep -m 10 "pattern" hugefile.log
# -F: faster (no regex)
# -m 10: stop after 10 matches

Advanced cat Techniques

Here Documents & Strings

$ cat << 'EOF' > script.sh
#!/bin/bash
echo "Hello $USER"
date
EOF

$ cat <<< "Quick inline text"

Here-doc for multi-line, Here-string for single line

Combining with tee

$ cat file1 file2 | tee combined.txt | wc -l
$ cat input.txt | tee -a log.txt | process_command
# tee: saves to file AND passes to next command

Creating Multi-File Archives

$ cat file1 file2 file3 > archive.tar
$ cat part1 part2 part3 > largefile.iso
# Can combine binary files too!

Interactive File Editing

$ cat >> diary.txt
# Type diary entry
Today I learned about grep and cat.
They are incredibly useful tools!
^D

$ cat diary.txt
Today I learned about grep and cat.
They are incredibly useful tools!

Real-World Examples

Log Analysis
$ grep "ERROR" app.log | cat -n
$ cat app.log | grep -c "404"
Configuration
$ cat /etc/passwd | grep "/bin/bash"
$ cat ~/.bashrc ~/.profile > bash_config.backup
Development
$ grep -r "TODO" src/ | cat -n
$ cat header.txt body.txt footer.txt > index.html
