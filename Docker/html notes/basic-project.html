Dockerfile Analysis

FROM node:18-alpine
WORKDIR /index
COPY package*.json .
RUN npm install
COPY . .
EXPOSE 3000
CMD ["npm", "start"]

Line by Line Explanation

FROM node:18-alpine

Uses Node.js 18 on Alpine Linux (lightweight ~50MB)

WORKDIR /index

Sets working directory inside container to /index

COPY package*.json .

Copies package.json and package-lock.json to working directory

RUN npm install

Installs dependencies listed in package.json

COPY . .

Copies all remaining project files to container

EXPOSE 3000

Documents that app listens on port 3000

CMD ["npm", "start"]

Default command to run when container starts

Key Insights

Layer Caching Strategy

Package.json copied separately before COPY . . to leverage Docker layer caching. Dependency installation layer won't rebuild unless package.json changes.

Alpine Advantage

Alpine Linux base image reduces image size significantly compared to regular Node image (~180MB vs ~1GB).

Convention vs Configuration

While WORKDIR /index works, convention suggests WORKDIR /app for better readability and standardization.

Package.json Analysis

Original (Error)

Missing Start Script 
{
  "name": "basic-project",
  "version": "1.0.0",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
    // ❌ Missing "start" script!
  },
  "dependencies": {
    "express": "^5.2.1"
  }
}

The Problem

Dockerfile runs CMD ["npm", "start"] but package.json has no "start" script. This causes the error: npm error Missing script: "start"

Fixed Version

Working 
{
  "name": "basic-project",
  "version": "1.0.0",
  "scripts": {
    "start": "node index.js",  // ✅ Added start script
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "dependencies": {
    "express": "^5.2.1"
  }
}

The Fix

Added "start": "node index.js" to scripts. Now npm start will execute node index.js as expected.

Important Concepts

Image vs Container

Docker Image

Blueprint or template

docker build -t basic-project .

Creates an image from Dockerfile

Docker Container

Running instance of an image

docker run -p 3000:3000 basic-project

Starts a container from the image

Name Relationships

package.json name

"name": "basic-project"

Only matters for npm publishing. Docker doesn't read this.

Docker image name

docker build -t express-demo .

Can be anything! Doesn't need to match package.json name.

express-demo amit-app mango-banana

Common Pitfalls & Solutions

Pitfall 1: Missing Start Script

Dockerfile runs npm start but package.json doesn't define it.

✅ Solution: Add "start": "node index.js" to package.json scripts

Pitfall 2: Express v5 Beta

Package.json uses Express 5 which is still in beta.

✅ Solution: Use Express 4 for production: "express": "^4.18.2"

Pitfall 3: Name Confusion

Thinking Docker image name must match package.json name.

✅ Solution: They're independent! Docker image can have any name.

Pitfall 4: File Name Mismatch

Renaming index.js but forgetting to update package.json start script.

✅ Solution: Keep start script filename in sync with your main file

Best Practices

Use .dockerignore

Create a .dockerignore file to exclude node_modules, logs, and other unnecessary files from the Docker build context.

node_modules npm-debug.log .git .env *.md

Standard WORKDIR

Use WORKDIR /app instead of /index for better standardization across projects.

Alternative CMD Syntax

Use CMD ["node", "index.js"] instead of CMD ["npm", "start"] to bypass package.json dependency.

Multi-stage Builds

For production, consider multi-stage builds to reduce final image size:

# Build stage FROM node:18-alpine AS builder WORKDIR /app COPY package*.json . RUN npm ci --only=production # Final stage FROM node:18-alpine COPY --from=builder /app/node_modules ./node_modules COPY . . CMD ["node", "index.js"]
