Service ‚Äî API Container

docker-compose.yml (API section)
api: build: . container_name: api ports: - "3000:3000" depends_on: - mongo-db networks: - devnet

‚úÖ api:

This is the service name. Used only inside the compose file for reference.

‚úÖ build: .

Meaning: Build image from Dockerfile

. = current folder
So: Docker looks for Dockerfile ‚Üí Builds image ‚Üí Uses it to run container

‚úÖ container_name: api

Instead of random name like api-project-api-1,
we FORCE container name to be simply: api

Much easier to reference and work with!

‚úÖ ports: "3000:3000"

Format:

HOST:CONTAINER

Meaning:

Your laptop port 3000 connects to container's port 3000

So you access API at: http://localhost:3000

‚úÖ depends_on: mongo-db

Meaning: Start mongo-db FIRST, then start api

API depends on database üíö

NOTE: This does NOT guarantee Mongo is fully ready, only ensures startup order. (Good enough for learning stage)

‚úÖ networks: devnet

THIS IS IMPORTANT!

It means: Put container inside network called devnet

Like connecting both machines to same WiFi üì∂

Result:

  • Containers can talk to each other
  • DNS works automatically
  • No IP addresses required

Your API can now connect to Mongo like this:

mongodb://mongo-db:27017

No localhost. No IP. Just the service name.

Service ‚Äî MongoDB Container

docker-compose.yml (MongoDB section)
mongo-db: image: mongo container_name: mongo-db volumes: - mongoData:/data/db networks: - devnet

‚úÖ image: mongo

We are NOT building anything here.

We simply pull the official image:

docker pull mongo

Then Docker runs it as a container.

‚úÖ container_name: mongo-db

So its container name becomes:

mongo-db

This is important because API uses this as hostname!

‚úÖ networks: devnet

Put Mongo inside same network as API.

Result:

  • They can see each other
  • They can communicate
  • mongo-db becomes reachable hostname

‚úÖ volumes: mongoData:/data/db

Purpose: Persistent storage

Without this, your data disappears when container stops!

mongoData ‚Üí Named Docker volume
/data/db ‚Üí Inside container MongoDB data location

Network Section

docker-compose.yml (Network section)

What This Does:

Creates a custom Docker network called devnet

Type = bridge network

(Default network type)
Provides automatic DNS resolution

Think of it like:

WiFi name = devnet
Both containers joined same WiFi

Communication Rules:

‚úÖ WORKS:

  • mongo-db ‚Üí api

‚ùå DOESN'T WORK:

How You Run It

Start Everything

$ docker compose up --build

This command does everything:

  • Builds API image from Dockerfile
  • Pulls MongoDB image
  • Creates devnet network
  • Starts all containers

Stop Everything

$ docker compose down

This command cleans up:

  • Stops all running containers
  • Removes containers (not volumes!)
  • Removes devnet network

MongoDB data persists in volume!

Useful Additional Commands

$ docker compose ps

List running containers

$ docker compose logs

View container logs

$ docker compose exec api bash

Enter API container shell

üí° Key Takeaways

Networking Made Easy
