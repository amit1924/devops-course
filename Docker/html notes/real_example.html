Part 1: Docker Run ‚Äì The Complete Breakdown

The Mystery: Why No Output?

Before (Foreground)

Terminal Session Node App Running
$ docker run -p 3000:3000 express-demo
Server running on port 3000
Listening at http://localhost:3000
  • Terminal attached to container
  • See output immediately
  • Terminal locked until Ctrl+C

After (Background)

Terminal Session Command Executed
$ docker run -d -p 3000:3000 --name express-app express-demo
c020f05d4326d45a8b9b1a3c7e8d4f2a
  • Container runs in background
  • Get container ID back
  • Terminal free for more commands

The Magic: Multiple Containers from One Image

Create 3 Containers from Same Image
$ docker run -d -p 3001:3000 --name express-1 express-demo
$ docker run -d -p 3002:3000 --name express-2 express-demo
$ docker run -d -p 3003:3000 --name express-3 express-demo
What's Happening?
  • express-demo ‚Üí Same image (blueprint)
  • Three containers ‚Üí Different names
  • Outside ports different (3001/3002/3003)
  • Inside app still runs on port 3000

ü§î Why Different Ports?

Because one port cannot be used twice on your laptop.
So we map differently: HOST:CONTAINER
Inside container, app still listens on 3000 ‚Äì and that's perfectly fine!

Part 3: Professional Image Versioning

Method 1: Build with Version (Recommended)

docker build -t express-demo:v1 .

Check Images:

$ docker image ls
REPOSITORY TAG IMAGE ID
express-demo v1 848126d4fc01

Method 2: Tag Existing Image

docker tag express-demo express-demo:v1

Your Actual Output (Perfect!):

express-demo:latest 848126d4fc01
express-demo:v1 848126d4fc01

Both have same IMAGE ID = They are the same image!
Just two different tags pointing to the same blueprint.

üéì What You Just Learned

docker tag express-demo express-demo:v1 means:
"Create another label called v1 pointing to this image."

Nothing was copied. No extra space used. Just another reference.

Running Different Versions Side-by-Side

Run Latest Version
