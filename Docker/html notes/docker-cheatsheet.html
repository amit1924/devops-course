What is Docker?

Containers

Think of containers as lightweight virtual machines that package your app with everything it needs to run.

Real example: When you run docker run nginx, you get a complete web server in seconds!

Images

Images are read-only templates used to create containers. Like a recipe for your app.

Your Dockerfile builds an image, then you create containers from it.

Layered Architecture

Docker images use layers. Each command in Dockerfile adds a layer. 

FROM node:18-alpine     ‚Üê Base layer (OS + Node)
COPY package.json .     ‚Üê New layer
RUN npm install        ‚Üê Another layer
COPY . .              ‚Üê Another layer
CMD ["npm", "start"]   ‚Üê Final layer

Installation & Verification

docker --version
Check Docker version installed
docker info

Pro Tip: Always run docker run hello-world after installation to verify everything works!

Image Management Mastery

docker images
List all downloaded images on your system
docker pull ubuntu:22.04
Download Ubuntu 22.04 image (specify version)
docker pull nginx:alpine
Download lightweight Nginx Alpine image
docker image inspect nginx
Get detailed JSON metadata about Nginx image
docker rmi nginx
Remove Nginx image (must stop containers first)
docker rmi $(docker images -q)
Remove ALL images (‚ö†Ô∏è Dangerous!)
docker image prune -a
Remove unused images (dangling images)
docker tag myapp:latest myapp:v1.0
Tag image with version number
docker history nginx
Show history/layers of Nginx image
Image Naming Convention: Use name:tag format like myapp:v1.2.3 or myapp:latest

Container Operations

Running Containers

docker run -d -p 8080:80 --name webserver nginx
Run Nginx in background, map host 8080 ‚Üí container 80
docker run -it ubuntu bash
Run Ubuntu interactively with bash terminal
docker run -d --name redis -p 6379:6379 redis
Run Redis server in background
docker run -d --name mysql -e MYSQL_ROOT_PASSWORD=pass -p 3306:3306 mysql
Run MySQL with environment variable
docker run -v /host/path:/container/path nginx
Run with volume mount (persistent storage)
docker run --rm alpine echo "Hello"
Run once and auto-remove after completion

Managing Running Containers

docker ps
List running containers
docker ps -a
List ALL containers (including stopped)
docker ps -q
Show only container IDs
docker ps --format "table {{.Names}}\t{{.Status}}\t{{.Ports}}"
Custom format output
docker stop container_name
Stop container gracefully (SIGTERM)
docker kill container_name
Force stop container (SIGKILL)
docker start container_name
Start stopped container
docker restart container_name
Restart container
docker pause container_name
Pause all processes in container
docker unpause container_name
Unpause container

Inspection & Debugging

docker logs container_name
View container logs
docker logs -f container_name
Follow logs (like tail -f)
docker logs --tail 100 container_name
Show last 100 lines of logs
docker logs --since 10m container_name
Show logs from last 10 minutes
docker exec -it container_name bash
Enter container with bash shell
docker exec -it container_name sh
Enter container with sh shell (Alpine)
docker exec container_name ls /app
Run command inside container without entering
docker top container_name
Show running processes in container
docker stats
Live resource usage (CPU, memory, network)
docker stats --format "table {{.Name}}\t{{.CPUPerc}}\t{{.MemUsage}}"
Custom stats format
docker inspect container_name
Detailed JSON info about container
docker inspect --format='{{.NetworkSettings.IPAddress}}' container_name
Get container IP address

Building Images

docker build -t myapp:latest .
Build image from Dockerfile in current directory
docker build -t myapp:v1.0 -f Dockerfile.prod .
Build with custom Dockerfile name
docker build --no-cache -t myapp .
Build without using cache (fresh build)
docker build --build-arg NODE_ENV=production .
Pass build arguments to Dockerfile

Sample Dockerfile

Dockerfile 
FROM node:18-alpine
WORKDIR /app
COPY package*.json .
RUN npm ci --only=production
COPY . .
EXPOSE 3000
USER node
CMD ["node", "index.js"]

Networks & Volumes

Network Commands

docker network ls
List networks
docker network create mynet
Create custom network
docker run --net=mynet --name app1 nginx
Run container in custom network
docker network inspect bridge
Inspect default bridge network
docker network connect mynet container_name
Connect container to network

Volume Commands

docker volume ls
List volumes
docker volume create db_data
Create volume
docker run -v db_data:/var/lib/mysql mysql
Use named volume for MySQL data
docker run -v $(pwd):/app node
Mount current directory to /app
docker volume prune
Remove unused volumes

Cleanup & System Management

docker rm container_name
Remove stopped container
docker rm -f container_name
Force remove running container
docker rm $(docker ps -aq)
Remove ALL containers (stopped)
docker system df
Show Docker disk usage
docker system prune
Remove all unused data
docker system prune -a --volumes
Remove everything (images, containers, volumes)
Warning: docker system prune -a will delete ALL images and containers. Use carefully!

Docker Compose Basics

docker-compose up
Start services defined in docker-compose.yml
docker-compose up -d
Start services in background
docker-compose down
Stop and remove services
docker-compose ps
List Compose services
docker-compose logs
View logs for all services
docker-compose build
Build images for services

Sample docker-compose.yml

docker-compose.yml 
version: '3.8'
services:
  web:
    build: .
    ports:
      - "3000:3000"
    volumes:
      - .:/app
  db:
    image: postgres:14
    environment:
      POSTGRES_PASSWORD: password
    volumes:
      - postgres_data:/var/lib/postgresql/data

volumes:
  postgres_data:

Practical Examples

1. Development Workflow

docker build -t myapp .
Build your app
docker run -d -p 3000:3000 --name myapp-container myapp
Run in background
docker logs -f myapp-container
Monitor logs
docker exec -it myapp-container sh
Debug inside container

2. Database Setup

docker run -d --name mysql-db -e MYSQL_ROOT_PASSWORD=secret -p 3306:3306 mysql:8
Run MySQL database
docker run -d --name redis-cache -p 6379:6379 redis
Run Redis cache

3. Web Server Setup

docker run -d --name nginx-proxy -p 80:80 -p 443:443 -v /nginx/conf:/etc/nginx nginx
Nginx with custom config
