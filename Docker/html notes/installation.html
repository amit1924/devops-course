Installation Steps

1

Remove Old Docker Versions

Completed

Clean up any existing Docker installations to prevent conflicts. 

sudo apt remove docker docker-engine docker.io containerd runc

Output Note

If you see "Package 'docker' is not installed", that's normal. It means no previous Docker installation was found.

2

Update System Packages

Completed

Ensure all system packages are up to date before installation. 

sudo apt update

sudo apt upgrade -y
3

Install Required Dependencies

Completed

Install certificates and tools needed for secure package management. 

sudo apt install -y ca-certificates curl gnupg
4

Add Docker's Official GPG Key

Completed

Add Docker's GPG key to verify package authenticity. 

sudo install -m 0755 -d /etc/apt/keyrings

curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg
5

Add Docker Repository (Critical Fix)

Important

Linux Mint 22.1 uses Ubuntu 24.04 (Noble) base. The repository must point to noble not xia.

Critical Issue Fixed

Docker doesn't recognize Mint's codename "xia". Use Ubuntu 24.04's codename "noble" instead.

Repository Command: 
echo \
"deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu \
$(. /etc/os-release && echo "$VERSION_CODENAME") stable" | \
sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
Manual Fix (if needed): 
sudo nano /etc/apt/sources.list.d/docker.list

# Change "xia" to "noble" in the file
# Save with Ctrl+O, exit with Ctrl+X
6

Install Docker Engine

Completed

Install Docker CE and all required components including Docker Compose. 

sudo apt update

sudo apt install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
7

Start and Enable Docker Service

Completed

Start Docker daemon and enable it to run on system boot. 

sudo systemctl enable docker

sudo systemctl start docker
8

Add User to Docker Group

Completed

Allow your user to run Docker commands without sudo. 

sudo usermod -aG docker $USER

Important Note

You need to log out and log back in for group changes to take effect. Alternatively, run newgrp docker in your current terminal.

9

Test Docker Installation

Completed

Verify Docker is working correctly with a test container. 

docker run hello-world

Success Output

Hello from Docker!
This message shows that your installation appears to be working correctly.

Portainer GUI Management

Portainer provides a web-based GUI for Docker management. This is the recommended interface for Linux Mint users. 

docker volume create portainer_data

docker run -d -p 9000:9000 --name portainer \
--restart=always \
-v /var/run/docker.sock:/var/run/docker.sock \
-v portainer_data:/data \
portainer/portainer-ce

Access Portainer

After installation, access Portainer at:

http://localhost:9000

Troubleshooting & Issues Fixed

Issue 1: MongoDB GPG Error

During sudo apt update, you encountered:

NO_PUBKEY 6A26B1AE64C3C388
The repository is not signed. 
sudo rm /etc/apt/sources.list.d/mongodb-org-6.0.list

sudo apt update

Issue 2: Docker Repo 404 Error

Docker doesn't recognize Mint codename "xia":

404 Not Found
The repository 'https://download.docker.com/linux/ubuntu xia Release' does not have a Release file.

Solution: Edit the Docker repository file and change xia to noble (Ubuntu 24.04).

Issue 3: Permission Denied on Docker Socket

After installation, running Docker gave:

permission denied while trying to connect to the Docker API at unix:///var/run/docker.sock

Solution: User wasn't in the Docker group. Fixed with sudo usermod -aG docker $USER and logging out/in.

Essential Docker Commands

Container Management

docker ps List running containers
docker ps -a List all containers
docker run -it ubuntu bash Run Ubuntu container
docker stop <container> Stop container

Image Management

docker images List images
docker pull nginx Pull Nginx image
docker rmi <image> Remove image
docker --version Check version
